# Установка докера в CorelDRAW 2018

Важное честное предупреждение: всё в папке `backend/` (HTML/CSS/JS/Python)
я проверил построчно по официальному справочнику API CorelDRAW 2018
(community.coreldraw.com/sdk/api/draw/20/...) и уверен в точности.

А вот сам механизм "создать докер и загрузить в него HTML-контрол" —
это VBA/GMS-код, который живёт внутри CorelDRAW и который я не могу
проверить без установленного CorelDRAW и его SDK под рукой. Поэтому ниже —
не готовый к вставке код, а точная инструкция, как взять этот код у Corel
и изменить в нём одну вещь.

## 1. Скачайте официальный шаблон

У Corel есть туториал с готовым шаблоном докера:

- Туториал: https://community.coreldraw.com/sdk/w/articles/174/custom-dockers-in-coreldraw-corel-designer-and-corel-photo-paint
- В нём — ссылка на `TemplateWebDocker.zip`

Внутри архива, как правило:

```
TemplateWebDocker/
├── Coreldrw.addon         (пустой файл-маркер — у нас уже есть свой, можно заменить)
├── AppUI.xslt             (регистрирует пункт меню/докер в интерфейсе)
├── UserUI.xslt            (задаёт, куда докер помещается по умолчанию)
├── TemplateWebDocker.gms  (VBA-проект: создаёт сам докер и HTML-контрол внутри)
└── TemplateWebDocker.html (пример содержимого докера — мы его заменим)
```

## 2. Скопируйте архив в Addons и переименуйте по вкусу

Папка адонов:

```
<папка установки CorelDRAW>\Programs64\Addons\AIAgentDocker\
```

Распакуйте туда содержимое архива.

## 3. Единственное обязательное изменение

Откройте `.gms`-проект через VBA IDE (Alt+F11 в CorelDRAW → найдите проект
в дереве). Там будет код, который создаёт докер и говорит HTML-контролу,
какой файл/адрес загрузить — обычно это путь к локальному `.html` файлу
рядом с `.gms`.

**Замените этот путь на:**

```
http://127.0.0.1:5055/
```

(порт можно сменить — см. `backend/app.py`, последняя строка).

Так докер будет открывать интерфейс, который раздаёт наш Python-backend, а
не статичный файл — все запросы (`/chat`, `/export_paths`...) станут
same-origin, и не будет проблем с CORS в старом IE-движке докера.

Остальной VBA-код шаблона (создание самого окна докера, его заголовок,
показ/скрытие) можно не трогать — вся логика чата и вызовы к CorelDRAW
теперь живут в `app.js`, который выполняется внутри этого HTML-контрола
через `window.external.Application`.

## 4. (Опционально) автозапуск backend вместе с CorelDRAW

Если хотите, чтобы `python app.py` стартовал сам при открытии CorelDRAW, в
VBA-проекте шаблона (в событии, которое срабатывает при загрузке аддона)
добавьте:

```vb
Shell "cmd /c cd /d ""C:\путь\к\coreldraw-ai-agent\backend"" && python app.py", vbHide
```

Это обычная функция VBA `Shell`, тут я уверен на 100% — она просто
запускает внешний процесс. Имеет смысл сначала проверять `/health`, чтобы
не плодить второй процесс, если backend уже запущен.

## 5. Если докер рендерится "как в 2009 году"

Встроенный HTML-контрол по умолчанию может стартовать в старом режиме
совместимости IE. Классический фикс для приложений, встраивающих
WebBrowser-контрол — добавить в реестр под
`HKEY_CURRENT_USER\Software\Microsoft\Internet Explorer\Main\FeatureControl\FEATURE_BROWSER_EMULATION`
значение DWORD `CorelDRW.exe` = `11001` (заставляет грузить в режиме IE11).
Имя exe-файла и точное значение стоит свериться под вашу сборку.

| Часть | Уверенность |

| Сам факт существования HTML-докеров и `window.external.Application` | Высокая, подтверждено официальным туториалом Corel |
| Точный VBA-код создания докера / содержимое AppUI.xslt/UserUI.xslt | Не проверено мной — берите из `TemplateWebDocker.zip` |
