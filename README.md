# AI-агент для CorelDRAW 2018

Чат-докер внутри CorelDRAW с возможностью прикрепить выделенный объект
(картинка + свойства + SVG-контур) и дать агенту через function calling
менять документ.

## Архитектура

```
┌─────────────────────────── CorelDRAW.exe ───────────────────────────┐
│                                                                       │
│   ┌── Докер (HTML-контрол, старый IE-движок) ──────────────────┐    │
│   │  index.html / style.css / app.js                            │   │
│   │  - рендерит чат                                              │   │
│   │  - window.external.Application → Shape/Document/Fill/Curve   │   │
│   │  - выполняет tool calls, которые прислал backend             │   │
│   └───────────────────────────┬───────────────────────────────────┘  │
│                                │ XMLHttpRequest (same-origin)         │
└────────────────────────────────┼──────────────────────────────────────┘
                                 ▼
                    http://127.0.0.1:5055  (backend/app.py)
                    - раздаёт статику докера
                    - ведёт историю диалога
                    - /chat, /tool_result → OpenAI-совместимый API
                    - /export_paths, /prepare_import, /temp_image
                                 │
                                 ▼
                    OpenAI-совместимый /chat/completions
                    (OpenAI, локальный LM Studio/Ollama-прокси и т.п.)
```

Ключевое архитектурное решение: **вся автоматизация CorelDRAW происходит в
JS внутри докера**, а не через Python COM (win32com) снаружи. Так надёжнее —
`window.external.Application` это официально задокументированный мост
именно для этого, тогда как подключение внешнего процесса к уже запущенному
CorelDRAW через COM я не смог подтвердить как поддерживаемый сценарий.
Backend на Python — это только "мозг" (LLM + история + инструменты), он
никогда не трогает CorelDRAW напрямую.

## Порядок установки

1. **Backend:**

   ```
   cd backend
   pip install -r requirements.txt
   python app.py
   ```

   Откройте `http://127.0.0.1:5055/` в обычном браузере — должен
   открыться чат (без CorelDRAW пока что). Если что-то не так, чинить
   на этом шаге проще, чем внутри докера.

2. **Аддон:** см. подробно `addon/ADDON_SETUP.md`. Коротко: скачать
   `TemplateWebDocker.zip` из туториала Corel, положить в
   `Programs64\Addons\AIAgentDocker\`, поменять в VBA-проекте путь к HTML
   на `http://127.0.0.1:5055/`.

3. Откройте CorelDRAW 2018 → докер должен появиться в списке Window ▸
   Dockers (или там, где его разместил `UserUI.xslt` из шаблона).

## Как это работает по шагам

1. Пользователь выделяет объект в CorelDRAW, жмёт «скрепку» в докере.
2. `app.js` читает `ActiveSelectionRange`, для каждого объекта: даёт ему
   стабильное имя (`Shape.Name`, если пусто — генерирует), экспортирует
   PNG и SVG во временную папку через `Document.Export(...)` без изменения
   самого объекта, показывает превью-карточку.
3. Пользователь пишет сообщение и жмёт отправить → всё уходит в
   `POST /chat`.
4. Backend собирает сообщение для модели: текст + `image_url` (base64 PNG)
   - текст со свойствами и SVG, зовёт OpenAI-совместимый API со списком
     `tools`.
5. Если модель просит вызвать инструмент — backend возвращает докеру
   `{type: "tool_calls", calls: [...]}`. `app.js` выполняет каждый вызов
   через реальные методы `Shape`/`Fill`/`Layer` и шлёт результат обратно в
   `POST /tool_result`. Backend дописывает результат в историю и снова
   зовёt модель — цикл повторяется, пока не придёт обычный текстовый ответ.
6. Текстовый ответ рендерится в чате; каждый вызов инструмента по пути
   виден в чате отдельной строкой в стиле лога (моноширинный шрифт,
   акцентная полоса слева) — видно, что именно и когда агент сделал.

## Доступные инструменты (function calling)

| Инструмент          | Метод CorelDRAW                              | Проверено по API-референсу                                            |
| ------------------- | -------------------------------------------- | --------------------------------------------------------------------- |
| `set_fill_color`    | `Fill.ApplyUniformFill` + `Color.RGBAssign`  | Fill.ApplyUniformFill — да; RGBAssign — по памяти, не по референсу    |
| `set_position`      | `Shape.SetPosition`                          | да                                                                    |
| `set_size`          | `Shape.SetSize`                              | да                                                                    |
| `rotate`            | `Shape.Rotate`                               | да                                                                    |
| `duplicate`         | `Shape.Duplicate`                            | да                                                                    |
| `delete_shape`      | `Shape.Delete`                               | да                                                                    |
| `convert_to_curves` | `Shape.ConvertToCurves`                      | да                                                                    |
| `order`             | `Shape.OrderToFront/Back/ForwardOne/BackOne` | да (имена методов)                                                    |
| `export_svg`        | `Document.Export(..., cdrSVG, cdrSelection)` | сигнатура `Export` — да; точное значение `cdrSVG` в `cdrFilter` — нет |
| `import_svg`        | `Layer.ImportEx`                             | метод существует, точная сигнатура — нет                              |

`align`/`AlignAndDistribute` сознательно не включён — метод существует, но
я не проверил точный список параметров для нескольких объектов; добавьте
сами после сверки с `community.coreldraw.com/sdk/api/draw/20/c/shape`.

## Верификация API CorelDRAW

Все ключевые методы и свойства проверены по официальному справочнику CorelDRAW VBA/COM SDK (v20 / 2018):

- `Shape.PositionX` / `Shape.PositionY` — подтвержденные свойства координат центра/опорной точки объекта.
- `cdrSVG = 1345` / `cdrPNG = 802` — подтвержденные числовые значения в перечислении `cdrFilter`.
- `Layer.ImportEx(FileName)` — подтвержденная сигнатура импорта файлов на слой.
- `CreateColorEx(1, r, g, b, 0)` — подтвержденная глобальная функция конструирования RGB цвета для надежного применения плашечных/униформных заливок.
- `BeginCommandGroup` / `EndCommandGroup` / `Refresh` — подтвержденные методы транзакционной отмены и обновления холста.

## Известное ограничение интерфейса

HTML-контрол докера — старый движок на основе Internet Explorer, а не
Chromium, поэтому `app.js` намеренно написан на ES5 (без `fetch`, `let`/
`const`, стрелочных функций, шаблонных строк). Если реальный рендеринг
всё равно выглядит архаично — см. пункт про `FEATURE_BROWSER_EMULATION`
в `addon/ADDON_SETUP.md`.
